## 📖 Project Overview

Customer behavior is one of the most valuable assets for retail businesses. Understanding how customers purchase products, respond to discounts, subscribe to services, and interact with different product categories can help organizations improve revenue, retention, and marketing effectiveness.

This project analyzes **3,900 customer purchase transactions** to uncover actionable insights into customer purchasing behavior, product performance, subscription trends, and revenue contribution across demographic segments.

The project follows a complete Data Analytics workflow:

**Python → PostgreSQL → Power BI → Business Report → Executive Presentation**

---

## 🎯 Business Objective

The primary objective of this project is to answer key business questions such as:

* Which customer groups generate the highest revenue?
* How effective are discounts in influencing purchase behavior?
* Which products receive the highest customer ratings?
* Are subscription members more valuable than non-subscribers?
* Which age groups contribute the most revenue?
* What customer segments should be targeted for future growth?

# 🐍 Phase 1: Exploratory Data Analysis (Python)

The project started with extensive Exploratory Data Analysis (EDA) using Python.

### Tools Used

* Python
* Pandas
* NumPy
* Matplotlib
* Seaborn

### Data Cleaning Activities

### Missing Value Treatment

The dataset contained missing values in the **Review Rating** column.

To preserve data quality and avoid unnecessary row deletion:

* Missing ratings were identified.
* Ratings were imputed using category-level median values.

### Column Standardization

Column names were standardized into snake_case format to improve readability and database compatibility.

### Feature Engineering

Two additional analytical features were created:

#### Age Group

Customers were grouped into:

* Young Adult
* Adult
* Middle-Aged
* Senior

#### Purchase Frequency Days

Created a numerical representation of purchase frequency to support behavioral analysis.

### Data Validation

A consistency check revealed redundancy between:

* Promo Code Used
* Discount Applied

The redundant field was removed before database loading.
